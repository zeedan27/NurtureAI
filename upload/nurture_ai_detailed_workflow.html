2
System
Phone number + OTP verification
Mother enters phone number — her own, or the CHW's if she doesn't own a phone. This is a critical rural BD decision: if a CHW's number is used, the CHW manages multiple families under one account. Supabase sends OTP. No email, no password.
If CHW's phone: CHW sees a family list; each child is a separate profile
3
Mother / CHW
Role selection
Two options shown as icon cards: Parent (single-child view, simplified UI) or Community Health Worker (multi-family batch list, additional clinical fields visible). Role determines which features and data fields appear throughout the entire app session.
4
Mother
Location setup via cascading dropdown
Division → District → Upazila selected from static dropdown lists. No GPS required, no live API call. This selection gates the facility database, community board content, EPI center locations, and local food suggestions shown throughout the app.
100% offline — upazila location data bundled at build time
5
Mother / CHW
Add child profile
Name, sex, and either a date of birth (existing child, postnatal modules activate) or expected due date (pregnant, prenatal module activates at -9 months). Birth weight and known conditions are optional — can be filled later or by CHW on a home visit.
6
System
Age auto-calculated — module routing
If EDD entered: age shown as gestational weeks, prenatal ANC guide and danger sign recognition activate. If DOB entered: age shown in weeks or months, postnatal care modules activate. At birth, when DOB is updated, profile transitions automatically from prenatal to postnatal — no manual switch.

~2MB total cache — works on entry-level Android

1
Mother
Opens app — online or offline
Dashboard loads instantly from service worker cache. If online, Supabase silently syncs any data written during the last offline session using a timestamp-based merge strategy. The mother sees no loading state either way.
2
System
Age recalculation + care card selection
Child's exact age recalculates from DOB on every app open. Correct age-band care card selected from cached JSON: 0–6m, 6–12m, 1–2y, 2–3y, 3–5y. Today's feeding cues, hygiene steps, and one age-appropriate stimulation activity rendered as an icon-first card.
3
Mother
Completes care card tasks
Taps each item on today's care card to mark done (feeding, hygiene, stimulation activity). Encouragement message shown on completion — no streaks, no failure states, no pressure for missed days. Low-anxiety design is deliberate for rural BD caregivers.
4
System
Vaccination badge check
EPI schedule scanned against child's current age and vaccination history. If any vaccine is due within 7 days and not yet marked received, a badge appears on the dashboard. Tapping it opens the full vaccination page for that specific vaccine.
5
System
WHO milestone nudge check
Milestone list checked against the child's current age band. If the child has just entered a new milestone window (e.g. turned 9 months) and the checklist has not been completed, a nudge card appears prompting the mother to run through the WHO milestone questions.
6
Mother
Optional: triage or myth chatbot
If the child seems unwell, mother taps "My child seems unwell" to enter the triage flow. If she has a health belief question ("Should I stop breastfeeding if I have a cold?"), she opens the Myth vs Fact chatbot from the bottom nav.
7
System
Background sync when connectivity resumes
All completed care tasks, triage episodes, and chatbot sessions persisted locally via IndexedDB. When device regains internet, Supabase sync runs in the background — no prompt to the mother, no data loss from offline sessions.
1
Mother
Taps "My child seems unwell"
Symptom screen opens. Child's current age in weeks/months is pre-loaded — age is the most critical input for triage (a 6-week-old with fever is a Level 4; a 3-year-old with the same fever may be Level 1). No internet needed to begin.
2
Mother
Icon-based symptom cards — tap to select
Six large icon cards with Bangla labels: fever / cough / diarrhea / rash / not eating / breathing difficulty. Tap to select, tap again to deselect. Multiple symptoms selectable. CHW mode adds a freetext "other" field for unusual symptoms not on the list.
3
Rule engine (local)
WHO IMCI rule table evaluates — offline
A static IMCI-based rule table runs locally on the device. Child age + symptom combination → pre-classification. Hard rules: breathing difficulty in any age = minimum Level 3. Breathing difficulty + fever in child under 2 months = automatic Level 4. Any single danger sign → Level 3 minimum. Rule engine output is the safety floor — Claude API cannot override it downward.
Fully offline — no API call at this step
4
Claude API
Claude validates and generates Bangla explanation
If online: structured JSON (child age in weeks, symptoms array, rule engine classification) is sent to Claude via LangGraph agent. Claude validates — it may raise the urgency level if the combination warrants it, but cannot lower a rule engine Level 3 or 4. Claude generates a 2–3 sentence plain-Bangla explanation of the likely cause and the specific action the mother should take right now.
If offline: rule engine output shown with pre-cached Bangla explanation text
5
System
Urgency card rendered — 4-way action branch
Color-coded card shown in Bangla. Action content is entirely different per level:
Level 1 — home care
Step-by-step home care steps in Bangla. A relevant myth/fact tip auto-pulled from cache based on the selected symptom.
Level 2 — visit CHW
Local CHW contact from upazila DB shown. One-tap generates a WhatsApp-formatted Bangla text with child age, symptoms, and urgency level to send ahead.
Level 3 — Upazila HC
Nearest UHC matched by capability (pediatric unit required). Bed count + oxygen status shown. If data is older than 24h: warning banner shown — "Call ahead to confirm."
Level 4 — district/tertiary
Full referral path: UHC → District hospital → Tertiary. Emergency contact numbers. PDF report auto-generated immediately for the mother to show at the facility.
6
Mother
Takes action — WhatsApp share for Level 2+
For Level 2 and above, the app generates a pre-filled WhatsApp message containing child's name, age, symptoms, and urgency level. Mother taps to send to the CHW or facility so they can prepare before arrival — no verbal description needed in a crisis.
7
Mother
Marks episode as resolved
Mother taps "Mark as resolved" once the child is better. Optional outcome note: home care worked / visited CHW / admitted to hospital / other. This closes the episode, stops the urgency card from showing on the dashboard, and triggers logging.
8
System
Episode auto-logged to medical history
Date, symptoms, urgency level, Claude explanation, action taken, and outcome note all appended to child's medical history. PDF report refreshes with the new episode. At the next facility visit, the doctor sees a complete episode log without needing a written diary from the mother.

Message in Bangla, naming the specific vaccine and the nearest EPI center.

Facility staff
Opens admin web URL in any browser
Completely separate URL from the parent app — no app store, no download. Works on any phone, tablet, or desktop at the facility. Staff members bookmark the URL and treat it like a daily shift tool. No IT installation needed.
2
System
Email + PIN authentication
Admin credentials created during facility onboarding by a district health administrator. Supabase row-level security ensures each facility account can only read and write its own facility record — no cross-facility data access possible.
3
Facility staff
Views current status + today's referral count
Dashboard shows: current published status (beds, oxygen, pediatric unit, power), time elapsed since last update, and today's count of Level 3/4 urgency triage cases directed here from the parent app. Referral count is aggregate only — no patient name, phone number, or PII shown.
4
Facility staff
Updates four availability toggles
Bed count (number input), oxygen available (yes/no), pediatric unit open (yes/no), generator/power backup (yes/no). One submit button. Designed to take under 30 seconds — simple enough for a reception clerk to run at each shift change without training.
5
System
Status published — live in parent app immediately
Supabase record updated. Any parent's Facility Finder that loads or refreshes now shows the updated bed count, oxygen status, and "Last updated X minutes ago" timestamp. No cache TTL delay — update is visible within seconds.
6
System
24-hour stale data warning auto-fires
If a facility has not submitted any update within 24 hours, every parent-facing facility card for that facility shows a warning banner: "Status may be outdated — call ahead to confirm availability." This prevents a mother from traveling to a full hospital based on stale capacity data — a patient safety guardrail, not a soft suggestion.
